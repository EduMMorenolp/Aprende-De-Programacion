Lección 8: Gestión de Proyectos con GitHub

Proyectos y Tableros en GitHub

GitHub proporciona herramientas para planificar y gestionar proyectos utilizando tableros. Puedes crear tareas, organizarlas en columnas y asignarlas a colaboradores.

Beneficios de los tableros de proyecto:

  • Visualización clara del flujo de trabajo.
  • Organización de tareas por prioridad y estado.
  • Asignación de tareas a colaboradores.
  • Seguimiento del progreso del proyecto.

Integración de Problemas y Solicitudes de Extracción en la Gestión de Proyectos

Puedes vincular problemas (issues) y solicitudes de extracción (pull requests) a elementos en los tableros de proyecto. Esto ayuda a mantener un seguimiento y una visibilidad claros de las tareas en curso.

Ejemplo de uso de problemas y solicitudes de extracción en un tablero:


// Crear un problema y asignarlo a un colaborador
En la página del repositorio en GitHub, ve a la pestaña "Issues".
Haz clic en "Nuevo issue" y proporciona los detalles del problema.
En el tablero de proyecto, arrastra el problema a la columna correspondiente.

// Vincular una solicitud de extracción a un problema
En la página de la solicitud de extracción, menciona el número del problema en el comentario.
La solicitud de extracción se vinculará automáticamente al problema y aparecerá en el tablero.

Organización y Etiquetas para Mantener el Orden

Utiliza etiquetas para categorizar y clasificar los problemas y las solicitudes de extracción. Esto facilita la búsqueda y permite identificar rápidamente el estado de cada elemento.

Ejemplo de uso de etiquetas:


// Crear una etiqueta
En la página del repositorio, ve a la pestaña "Issues" y haz clic en "Etiquetas".
Haz clic en "Nueva etiqueta" y proporciona un nombre y color.

// Aplicar etiquetas a problemas y solicitudes de extracción
Al crear o editar un problema o solicitud de extracción, puedes asignar etiquetas relevantes.
Por ejemplo, etiquetas como "bug", "mejora" o "pendiente" para identificar el tipo o estado.

Documentación y Colaboración en la Wiki del Repositorio

La wiki del repositorio en GitHub permite crear y mantener documentación relacionada con el proyecto. Puedes colaborar en la creación de guías, tutoriales y otros recursos útiles.

Ejemplo de colaboración en la wiki:


// Acceder a la wiki del repositorio
En la página del repositorio en GitHub, ve a la pestaña "Wiki".
Puedes crear nuevas páginas o editar las existentes directamente en línea.

// Colaboración en la documentación
Invita a colaboradores al repositorio para que puedan contribuir a la wiki.
Utiliza la sintaxis de formato Markdown para dar formato a los contenidos y añadir enlaces y recursos.
